# Processed DML Outputs

This directory holds the final, compiled bulk SQL `INSERT` transaction queries generated by our Python transformation code blocks. 

## 🚨 CRITICAL PREREQUISITE
Before running any of the data insertion files in this folder, you **must** execute the master layout script located in the `1.DDL` folder (`DDL-file.sql`). The database and empty table shells must exist before data can be loaded into them.

## ⚠️ MANDATORY Order of Execution
Because the tables are bound together by strict Relational Foreign Key constraints, running scripts out of sequence will trigger dependency crashes. You **must** execute these data inserts in the following exact sequence:

1. 🟩 **`province_names-dml.sql`** (Must go first; contains primary key codes used by patients).
2. 🟩 **`doctors-dml.sql`** (Must go second; contains reference codes used by admissions).
3. 🟩 **`patients-dml.sql`** (Must go third; relies on province codes to exist).
4. 🟩 **`admissions-dml.sql`** (Must go last; relies on both patient IDs and doctor IDs to successfully bind).
